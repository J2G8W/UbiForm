Detailed Description

The central concept to UbiForm. Each process should only need one component at any point and all network operations should go via the component (using the getters we access the actual function we want). The component is also used for creation, access and destruction of endpoints and controls the memory access of these objects.

◆ registerStartupFunction()

void Component::registerStartupFunction ( const std::string &  endpointType,
endpointStartupFunction  startupFunction,
void *  startupData 
)

This can be used to run the given function whenever the given endpoint is CONNECTED (that is it dials or listens). It runs as a std::thread in the background and is cleanly handled IF the given the function does not block on non-network calls.

Additionally the function should not throw any exceptions as these are incredibly difficult to diagnose, so the suggestion is to wrap any work in try/catch for good debugging If there is not an endpointType in our manifest then this function does nothing and if there is already a function registered then we overwrite this.

Parameters
endpointType- The type of endpoint we want to attach to
startupFunction- The function we want to call whenever the endpoint connects. It receives a pointer to the created endpoint and the startupData which is provided
startupData- Some arbitrary data which is passed as extra to the startup function (can be nullptr)
