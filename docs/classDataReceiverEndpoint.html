Detailed Description

This class represents a data receiver in our design. I have made the choice that it is only able to dial, not listen as this fits with the IOT scope best. The receive of messages is the same no matter the type of endpoint BUT our dialing process is different, hence why this is extended in the child classes

We receive a message asynchronously, accepting a function which does work a EndpointMessage. The function handles the memory management of the EndpointMessage. Additionally we are able to pass in arbitrary data as additionalData which is then accessible as the second attribute in the called function

Parameters
callback- The function which is called when a EndpointMessage is received - DON'T BE A BLOCKING FUNCTION or we can get deadlock scenarios
furtherUserData- Extra data which you want to be available in the call back
Exceptions
SocketOpenError- When the endpoint is not open
